## 1. Executive Summary

NeuralTwin은 오프라인 리테일 매장의 고객 행동을 IoT 센서로 수집하고, AI로 분석하며, 3D 디지털 트윈으로 시뮬레이션하는 **B2B SaaS 플랫폼**입니다. 온라인 커머스가 모든 클릭을 추적하듯, 오프라인 매장에서도 동일한 수준의 데이터 기반 의사결정을 가능하게 합니다.

### 1.1 현재 제품 구현 현황 (2026년 3월 기준)

| 구성 요소 | 구현 완료 ✅ | 미완성 — 우선 개발 필요 🔴 |
|-----------|------------|--------------------------|
| **IoT / NeuralSense** | Pi Sniffer, MQTT, 삼변측량, 캘리브레이션, 6종 JSONL 출력 | 🔴 Supabase 브릿지, MQTT TLS, 원격 배포 자동화 |
| **Backend (Edge Functions)** | 53개 EF, 153개 테이블, AI 분석 파이프라인 | 🔴 RLS 정책, CI/CD, service_role 분리, CORS 중복(53회) 제거 |
| **OS Dashboard** | 3D 엔진, 16개 오버레이, 60+ 컴포넌트, KPI 대시보드 | 멀티 매장 비교 뷰, 월간 PDF 리포트, 실시간 푸시 알림 |
| **Website** | 랜딩, AI 챗봇(인라인+풀스크린), 리드 캡처, 3개국어 | 블로그, 케이스 스터디, 가격 페이지 |
| **AI 어시스턴트** | retail-chatbot (26파일), neuraltwin-assistant (34파일) | @neuraltwin/ai-core 패키지, A/B 테스트 프레임워크 |

> **🔴 최우선 기술 과제**  
> NeuralSense(IoT) → Supabase 연결 브릿지(`supabase_bridge.py`)가 미완성 상태입니다.  
> 이 브릿지 없이는 실시간 대시보드 데모가 불가능하며 모든 SaaS 매출이 막혀 있습니다.  
> **Phase 1 Week 1~2에 이 작업이 절대 최우선입니다.**

NeuralTwin의 고객은 단일 의사결정자가 아닙니다. **구매(Buyer) / 사용(End User) / 내부 전파(Champion)**의 세 층이 각각 다른 언어로 설득되어야 합니다.

### 2.2 페르소나별 소구 포인트

#### 🏢 페르소나 A: 김운영 — 프랜차이즈 리테일운영팀장 (Primary Buyer)

| 항목 | 내용 |
|------|------|
| **역할** | 50개 매장의 운영 성과 관리, 매장 표준화 담당 |
| **고통 (Pain)** | 매장별 성과 편차가 큰데 원인을 모름. 점장은 "감"으로 답변 |
| **목표 (Goal)** | 성과 좋은 매장의 패턴을 데이터로 찾아 전체에 복제 |
| **구매 기준** | ROI 증명 > 도입 용이성 > 가격 |
| **핵심 메시지** | *"50개 매장을 한 화면에서, AI가 차이점을 분석합니다."* |

#### 🏪 페르소나 B: 박점장 — 매장 점장 (End User)

| 항목 | 내용 |
|------|------|
| **역할** | 단일 매장 운영, 직원 관리, 매출 책임 |
| **고통 (Pain)** | 어떤 진열이 효과적인지 알 수 없음. 본사 지시가 현장과 안 맞아도 근거가 없음 |
| **목표 (Goal)** | 내 매장 흐름을 이해하고, 본사에 데이터로 의견 제시 |
| **핵심 메시지** | *"오늘 매장에서 무슨 일이 일어났는지, 숫자로 보세요."* |

#### 📊 페르소나 C: 이데이터 — 본사 데이터/마케팅 담당자 (Champion)

| 항목 | 내용 |
|------|------|
| **역할** | 온오프라인 데이터 통합 분석, 캠페인 효과 측정 |
| **고통 (Pain)** | 온라인은 GA4로 다 보이는데, 오프라인은 POS 매출 숫자만 존재 |

### 2.3 실명 타겟 고객 & 영업 전략

#### Tier 1 — 즉시 접근 (3개월 내 매출 목표)

---

**🎯 CJ 올리브영 (1,300+ 매장)**

- **왜 지금인가:** 2024년 매출 4.5조원. 이미 데이터 기반 문화 강함. 1,300개 매장 레이아웃 최적화 근거 없음
- **접근 Entry Product:** 매장 진단 리포트 (강남/명동 플래그십 1곳)
- **영업 메시지:** *"올리브영N 성수의 1,400평 동선을 데이터로 최적화하면, 그 패턴을 1,300개 매장에 복제할 수 있습니다"*
- **예상 ACV:** 진단 ₩500만 → Growth SaaS 전환 시 **50매장 × ₩299K × 12 = ₩1.8억/년**

---

**🎯 아성다이소 (1,500+ 매장)**

- **왜 지금인가:** 2024년 매출 3.97조원. 뷰티 카테고리 매출 144% 급증 → 뷰티 존 최적화 급선무. 충동구매 비율 높아 동선 설계가 매출에 직결
- **접근 Entry Product:** 매장 진단 리포트 (뷰티 존 특화)
- **영업 메시지:** *"뷰티 코너의 위치와 배치에 따라 고객 체류시간이 최대 2배 차이납니다"*
- **예상 ACV:** 진단 ₩400만 → **100매장 × ₩99K × 12 = ₩1.2억/년**

---

**🎯 무신사 스탠다드 / 무신사 뷰티 (오프라인 확장 중)**

- **왜 지금인가:** 2026년 성수·홍대에 300~400평 뷰티 전용 매장 오픈 예정. 온라인 네이티브 → 처음부터 NeuralTwin을 매장 설계에 포함 가능
- **접근 Entry Product:** 매장 진단 리포트 + SaaS 동시 제안 (신규 매장)
- **영업 메시지:** *"온라인에서 퍼널 분석하듯, 오프라인 입장→체류→구매 전환을 실시간으로 보여드립니다"*
- **예상 ACV:** **3매장 × Enterprise ₩599K × 12 = ₩2,500만/년**

## 5. 세일즈 전략 & 플레이북

### 5.1 세일즈 프로세스

```
Discovery → Demo → PoC → Proposal → Close → Onboard
   1주       1주    2주     1주      1~2주    1주

총 세일즈 사이클: 6~8주
PoC → 유료 전환율 목표: 60%
유료 진단 → SaaS 전환율 목표: 40%
```

| 단계 | 활동 | 성공 기준 |
|:----:|------|---------|
| **1. Discovery** | 니즈 파악, BANT 검증 | Budget/Authority/Need/Timeline 확인 |
| **2. Demo** | 맞춤 3D 데모 (업종별 3종) | "이런 데이터가 필요했다" 반응 |
| **3. PoC** | 2주 파일럿: Pi 4대 설치, 실데이터 수집 | 안정성 95%+, 실행 가능 인사이트 3개+ |
| **4. Proposal** | 견적/제안서, ROI 계산기로 수익 증명 | 의사결정자 ROI 납득 |
| **5. Close** | 계약 체결 | 계약서 서명 |
| **6. Onboard** | Pi 설치, 7단계 위자드, 첫 인사이트 확인 | 첫 30일 성공 기준 리뷰 완료 |

### 5.2 데모 시나리오 3종

#### 시나리오 A: 패션 브랜드 본사
- **타겟:** 리테일운영팀장
- **핵심 Hook:** *"상위 매장의 패턴을 3D로 보여드리고, 하위 매장에 적용 시뮬레이션을 돌려보겠습니다"*
- **데모 흐름:** 멀티 매장 KPI 비교 → AI가 차이점 분석 → 3D Best Practice 레이아웃 시각화

#### 시나리오 B: 뷰티 매장 점장
- **타겟:** 매장 점장/MD
- **핵심 Hook:** *"지난 주 진열 변경 효과를 숫자로, 다음엔 변경 전에 시뮬레이션을 돌려보세요"*
- **데모 흐름:** 진열 변경 전후 존별 체류시간 비교 → 전환율 변화 → ROI 자동 계산

#### 시나리오 C: 백화점 운영팀
- **타겟:** 임대/운영팀장
- **핵심 Hook:** *"임대료 협상에서 데이터가 있는 쪽이 이깁니다"*
- **데모 흐름:** 층별·시간대별 트래픽 히트맵 → 테넌트별 기여도 → 최적 믹스 시뮬레이션

#### Play 2: "온라인처럼 보여드립니다" — 데이터 네이티브 기업 공략

**타겟:** 무신사, 마켓컬리 오프라인, 29CM 쇼룸 등 온라인 출신 기업

> *"온라인에서는 GA4로 페이지뷰, 체류시간, 전환율을 다 보시잖아요. 오프라인 매장에서도 똑같이 보여드립니다.*
>
> *매장 입구 = Landing Page (바운스율)*  
> *각 존 = 페이지 (체류시간, 스크롤 깊이)*  
> *피팅룸/계산대 = 전환 이벤트*
>
> *AI한테 '오늘 왜 전환율이 떨어졌지?' 물어보세요. AI가 '비가 와서 유동인구가 줄었고, A존 체류시간이 평소 대비 40% 감소했다'고 답해줍니다."*

이 고객들은 데이터의 가치를 이미 알고 있어 **설득이 아니라 "오프라인에도 같은 것이 있다"는 존재 인식만 시키면** 됩니다.
