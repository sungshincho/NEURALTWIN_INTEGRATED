## 1. 경쟁사 심층 분석

### 1.1 RetailNext

**개요**: RetailNext는 세계 최대 인스토어 애널리틱스 플랫폼으로, Aurora 올인원 센서를 핵심 하드웨어로 사용합니다. 250개 이상의 리테일러가 50개국 이상에서 도입 중입니다.

**2025-2026 최신 기능**:
- **Aurora 센서**: 하루 수백만 개의 데이터 포인트를 캡처하는 올인원 센서. 센서끼리 통신하여 매장 전체의 shopper journey를 단일 뷰로 구축
- **Retail Media Network ROI 측정** (2025년 핵심 포커스): 검증된 노출(verified impressions) 기반 미디어 가격 책정, 실시간 점유율 모니터링으로 동적 디지털 사이니지 콘텐츠 최적화
- **예측 분석(Predictive Analytics)**: 과거 분석을 넘어 알고리즘이 쇼퍼 행동, 미디어 참여, 전환율 패턴을 파악하여 미래 성과 예측
- **자산 보호(Asset Protection)**: 클라우드 기반 도구로 ORC(Organized Retail Crime) 조사 시간을 며칠에서 시간 단위로 단축
- **NRF 2026 참가**: 2026년 NRF에서 최신 파트너십 및 기술 공개

**가격 정책**:
- **커스텀 견적 기반** (공개 가격 없음)
- 센서 단위/월 구독 모델 (per-sensor, per-month)
- 사용자 수, 리포팅, API 추가 비용 없음
- G2 리뷰에 따르면 "매우 비싼 카메라, 기능을 제대로 쓰려면 월 수천 달러의 소프트웨어 구독"
- **추정가**: 센서 1대당 월 $200-300 수준으로 업계에서 추정 (공식 미확인)

**한국 진출 여부**:
- 한국어를 포함한 15개 언어 지원 (한국어, 일본어, 중국어 등)
- 그러나 **한국 내 전담 지사 또는 파트너 없음** (검색 결과 기준)
- 한국 시장에 특화된 영업 활동은 확인되지 않음
- NeuralTwin에게 한국 시장 선점 기회가 있음

### 1.2 Sensormatic Solutions (Johnson Controls)

**개요**: Johnson Controls 산하의 리테일 솔루션 사업부. 2026년 60주년을 맞이하며 손실 방지, 트래픽 인사이트, 재고 인텔리전스의 통합 생태계를 제공합니다.

**2025-2026 최신 동향**:
- **Orbit AI 센서** (2025년 12월 발표): AI 기반 오버헤드 People Counter에 Re-Identification(Re-ID) 기술 탑재
  - 개인 식별 정보(PII) 없이 직원과 비쇼퍼를 자동 제외
  - 쇼퍼 인구통계, 이탈률, 쇼퍼 전용 전환율, 체류시간, 이동 패턴 분석
  - 존별 선호도 및 체류시간 데이터로 디스플레이 효과 측정
- **Video AI 카메라**: Re-ID 기능이 Orbit AI 센서와 Video AI 카메라 양쪽에서 활용
- **올인원 서비스 모델** (2025년 5월 발표): 소스 태깅, 라벨링, RFID 하드웨어, TrueVUE Cloud 소프트웨어, 전문 서비스를 통합한 원스톱 패키지
- **NRF 2026**: "The AI Advantage: Turning Foot Traffic into Retail Intelligence" 세션 주관

**가격 정책**: 공개되지 않음. 엔터프라이즈 레벨 커스텀 견적

**NeuralTwin 대비 평가**: Sensormatic는 대형 유통사 대상의 종합 솔루션이며, NeuralTwin의 타겟인 중소형 프랜차이즈/단일 매장 세그먼트와 직접 경쟁 가능성은 낮음. 다만, 3D 디지털 트윈과 자연어 AI 질의 기능은 Sensormatic에 없는 NeuralTwin만의 차별점

**현재 상태**: ShopperTrak는 2015년 Tyco(현 Johnson Controls)에 $175M에 인수된 후, 현재 Sensormatic Solutions의 트래픽 인사이트 브랜드로 완전 통합 운영 중입니다.

**핵심 수치**:
- 전 세계 100개국 이상, **150만 개 이상의 센서** 설치
- 연간 **400억 건 이상의 쇼퍼 방문** 데이터 보유
- 25년 이상의 리테일 트래픽 인사이트 경험

**독립 브랜드로서의 ShopperTrak은 사실상 소멸**. Sensormatic ShopperTrak으로 명칭이 통합되었으며, Sensormatic IQ 운영 플랫폼의 일부로 제공됩니다.

**NeuralTwin 시사점**: ShopperTrak의 주요 기능(트래픽 카운팅)은 NeuralTwin의 기본 기능에 해당하며, NeuralTwin은 이를 넘어서 AI 인사이트, 3D 시뮬레이션까지 제공하는 차별화 가능

### 1.4 loplat (로플랫)

**개요**: 한국 오프라인 데이터 전문 기업. WiFi/Bluetooth/GPS 기반 위치 인식 SDK를 모바일 앱에 내장하여 오프라인 방문 데이터를 수집합니다.

**2025-2026 최신 동향**:
- **2025년 아기유니콘** 선정 (AI/빅데이터 분야)
- 월 약 **700만 MAU**, 월간 약 **20억 건의 위치 데이터 이벤트** 처리
- 60개 이상의 뷰티 브랜드, 500여 종의 상품 데이터와 연계
- WiFi 신호 기반 실내 위치 인식 기술 10년 경험 보유

**주요 제품군**:
| 제품 | 설명 |
|------|------|
| **loplat X** | SDK 기반 실시간 위치 인식 + 위치 기반 마케팅 (푸시 알림) |
| **loplat I** | 오프라인 고객 여정 분석 대시보드 |
| **loplat DMP** | 위치 기반 마케팅 타겟 세그먼트 제공 |

**가격**: 공개되지 않음. B2B 맞춤 견적 (business@loplat.com 문의)

**NeuralTwin과의 핵심 차이**:

| 기준 | NeuralTwin | loplat |
|------|-----------|--------|
| 데이터 수집 방식 | 자체 IoT 센서 (Pi) 설치 | 모바일 앱 SDK 내장 |
| 분석 범위 | 매장 내부 구역별 동선/체류 | 매장 방문 여부 (in/out) |
| 앱 의존성 | 앱 불필요 | 앱 설치 필수 |
| 데이터 정밀도 | 구역 단위 (zone-level) | 매장 단위 (store-level) |
| 3D 시뮬레이션 | 있음 | 없음 |
| AI 자연어 분석 | 있음 | 없음 |

**NeuralTwin 시사점**: loplat과는 직접적 기능 경쟁보다는 **보완 관계**에 가까움. loplat은 "이 고객이 우리 매장에 왔는가?" 수준, NeuralTwin은 "매장 안에서 어디로 갔고 얼마나 머물렀는가?" 수준. 다만, loplat이 매장 내 정밀 분석으로 확장할 가능성은 모니터링 필요

**개요**: 글로벌 피플 카운팅 전문 기업. 2025년 3D Pro2 모델을 출시하며 AI 기반 매장 분석 기능을 강화하고 있습니다.

**2025-2026 주요 기능**:

- **Visitor Profile Metrics**: 방문객 수, 그룹 크기 감지, 평균 체류시간, 존별 히트맵

## 4. 기술 트렌드

### 4.1 WiFi RSSI 기반 실내 측위 최신 동향 (2025-2026)

**현 상태**: 2025년 기준, 인프라 없이 작동하는 보편적이고 정확한 실내 측위 시스템은 아직 없습니다. 그러나 WiFi 기반 측위는 기존 인프라를 활용할 수 있어 가장 유망한 접근 방식입니다.

**정확도 개선**:
| 기술 | 정확도 | 비고 |
|------|:------:|------|
| WiFi RSSI 기본 | 3~5m | 환경 변수에 민감 |
| WiFi RTT (802.11mc) | **1~2m** | Android 9+ 지원. 신호 왕복 시간 기반으로 RSSI보다 환경 간섭에 강함 |
| Hybrid WiFi RTT-RSS | **0.6m** | 2025년 연구 결과. 3개 실제 환경 테스트 |
| ML 기반 핑거프린팅 | 1~3m | 그래프 신경망 + 시간 합성곱 네트워크 활용 |

**주요 기술 발전**:
1. **IEEE 802.11az-2022**: 802.11mc의 후속 표준. Fine Timing Measurement(FTM) 개선
2. **IEEE 802.11bk-2025**: 최신 표준. 6GHz 대역 320MHz 채널 지원으로 RTT 정확도 향상
3. **머신러닝 통합**: 핑거프린팅 기술에 ML을 적용하여 환경 변화 시 재캘리브레이션 없이 위치 정확도 유지
4. **하이브리드 접근**: RSS + RTT + 관성 센서를 결합한 다중 모달 측위

**NeuralTwin 기술 시사점**:
- RSSI 기본 정확도(3~5m)는 **zone-level 분석에 충분** (매장 내 구역 = 보통 3~10m)
- RTT 기반으로 업그레이드하면 1~2m까지 개선 가능하나, AP 교체 필요
- ML 기반 핑거프린팅은 Pi 센서의 RSSI 데이터와 자연스럽게 결합 가능
- **로드맵 제안**: Phase 1은 RSSI(현재), Phase 2에서 ML 핑거프린팅 도입, Phase 3에서 RTT 하이브리드 고려

### 4.2 MAC 랜덤화 대응 기술

**문제**: iOS 14+와 Android 10+ 이후 MAC 주소 랜덤화가 기본 활성화되어, WiFi Probe Request의 MAC 주소로 고유 사용자를 식별하기 어려워졌습니다.

**최신 대응 기술 (2024-2025 연구)**:

| 기술 | 정확도 | 원리 |
|------|:------:|------|
| **시간-주파수 패턴 분석** | **최대 99%** | Probe Request 발신의 시간-주파수 패턴이 기기마다 고유. 2024년 8월 논문 |
| **Information Element 핑거프린팅** | 90%+ | Probe Request에 포함된 IE 필드로 기기 식별 |
| **타이밍 공격** | 높음 | 네트워크 스캔 타이밍의 inter-frame arrival time으로 기기 그루핑 |
| **멀티채널 스니핑 + 2단계 클러스터링** | 높음 | Probe Request의 콘텐츠와 발신 속도를 결합한 클러스터링 |
| **Bleach 알고리즘** | — | WiFi probe-request 시그니처에서 MAC 연관 추출 (2024년 논문) |

**핵심 인사이트**: "MAC 랜덤화만으로는 프라이버시가 보장되지 않는다"는 것이 연구 커뮤니티의 합의입니다. 핑거프린트와 시간 정보만으로도 랜덤화를 우회할 수 있습니다.

**IEEE 대응**: IEEE 802.11aq Task Group이 Randomized and Changing MAC (RCM) 운영 표준화 작업 진행 중

**NeuralTwin 기술 시사점**:
- MAC 랜덤화는 **완전한 방문객 카운팅 정확도 보장이 불가능**하게 만들지만, 대응 기술이 빠르게 발전 중
- NeuralTwin의 RSSI 패턴 기반 재방문 추론 알고리즘(NT_Master 기재)은 학술적으로도 유효한 접근
- **Probe Request의 IE 필드 + 타이밍 패턴 분석**을 NeuralSense에 추가하면 정확도 크게 향상 가능
- 고객에게는 "100% 정확한 고유 방문객 수가 아닌, **트래픽 패턴과 트렌드 분석**에 초점"이라는 메시지가 적절

### 4.5 한국 개인정보보호법(PIPA)과 WiFi 센싱

**PIPA 2025-2026 주요 업데이트**:

| 시행일 | 내용 |
|--------|------|
| 2025.03 | **데이터 이동권(Data Portability)**: 개인이 자신의 데이터를 다른 서비스로 이전 요청 가능 |
| 2024.12.31 | **통합 개인정보 처리 가이드라인** 초안 발표 (개정 PIPA 반영) |
| 2025.08.06 | **생성형 AI 가이드라인** 발표 (PIPC) |
| 2026.01 | **인공지능 개발 및 신뢰성에 관한 기본법** 시행 (2025.01.21 제정) |

**WiFi 센싱과 개인정보보호 - GDPR/PIPA 핵심 원칙**:

1. **MAC 주소는 개인정보**: GDPR 하에서 MAC 주소는 "자연인을 특정할 수 있는" 개인정보로 분류. PIPA도 유사 기준 적용
2. **해싱만으로 불충분**: 해싱/암호화된 MAC도 단일 기기를 특정할 수 있으면 **가명정보(pseudonymous data)**로 취급됨
3. **합법적 처리 근거 필요**: 동의 또는 정당한 이익(legitimate interest) 근거 필요
4. **일별 다른 해시 적용**: 일부 기업은 매일 다른 해시를 적용하여 날짜 간 상관관계 방지
5. **각국 DPA 감시 강화**: 벨기에, 영국, 네덜란드 등 DPA가 WiFi 트래킹을 적극 감시

**NeuralTwin 필수 조치 (PIPA 준수)**:

| 조치 | 우선순위 | 설명 |
|------|:--------:|------|
| MAC 해싱 + 솔트 + 절단 | 필수 | 원본 MAC이 플랫폼에 도달하기 전 처리 |
| 일별 해시 키 변경 | 권장 | 날짜 간 개인 추적 방지 |
| 데이터 익명화 | 필수 | 구역별 집계 데이터만 저장, 개별 동선 미저장 |
| 매장 내 안내문 | 필수 | "WiFi 신호를 이용한 매장 내 인원 분석 중" 고지 |
| 데이터 보존 기한 | 필수 | 목적 달성 후 즉시 삭제 (PIPA 제21조) |
| DPIA 실시 | 권장 | 개인정보 영향평가 (대규모 처리 시) |
| AI 기본법 준수 | 2026필수 | AI 활용 관련 신뢰성 확보 조치 |

**중요 참고**: PIPA에서 WiFi 센싱에 특화된 가이드라인은 아직 발표되지 않았습니다. 그러나 GDPR의 WiFi 트래킹 가이드라인과 PIPA의 일반 원칙을 결합하여 대비하는 것이 안전합니다.
