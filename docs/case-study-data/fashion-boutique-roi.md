## 2. 도입 배경 및 과제

### 기존 문제점

1. **피팅룸 병목 현상**: 주말/저녁 피크 타임에 피팅룸 대기 시간 최대 20분, 고객 이탈 발생
2. **VMD 감각 의존**: 매장 디스플레이 배치를 매니저의 경험과 감에 의존, 데이터 기반 의사결정 부재
3. **직원 배치 비효율**: 전 시간대 균등 배치로 피크 타임 인력 부족, 비피크 타임 유휴 인력
4. **전환율 정체**: 방문자 대비 구매 전환율이 업계 평균(3.5%) 이하로 정체

### 도입 목표

- 피팅룸 대기 시간 50% 감소
- 데이터 기반 VMD 최적화로 전환율 향상
- 시간대별 인력 배치 최적화
- 월매출 20% 이상 성장

## 4. NeuralTwin 활용 기능 상세

### 4.1 히트맵 동선 분석 → VMD 재배치

**적용 내용**:
- WiFi Probe 센서 6대 설치, 고객 동선 실시간 수집
- 히트맵 분석으로 "데드존" 3곳 식별 (Z3 좌측 코너, Z5 후면, Z2-Z3 경계)
- AI 추천 기반 VMD 재배치: 인기 상품을 데드존으로 이동, 동선 유도 사이니지 배치

**결과**:
- 데드존 방문율: 평균 8% → 22% (+175%)
- Z5 악세서리존 매출: +34% (크로스셀링 동선 유도 효과)
- 매장 전체 체류시간: 22분 → 28분 (+27%)

### 4.2 AI 피팅룸 대기열 관리 → 알림 시스템

**적용 내용**:
- 피팅룸 구역(Z4) 전용 센서로 실시간 점유 현황 모니터링
- 대기 고객에게 카카오톡 알림 발송 (예상 대기 시간, 차례 알림)
- 대기 중 Z5 악세서리존 방문 유도 추천

**결과**:
- 피팅룸 대기시간: 12분 → 6.5분 (-46%)
- 대기 중 이탈율: 23% → 8% (-65%)
- 대기 중 추가 구매율: 5% → 14% (피팅 대기 중 악세서리 구매)

### 4.3 시간대별 인력 배치 최적화

**적용 내용**:
- 시간대별 방문자 패턴 분석 (30일 데이터 기반)
- AI 최적 인력 배치 스케줄 생성 (주간/주말 분리)
- 피크 타임(17:00-20:00) 인력 집중, 비피크 타임 최소 운영

**결과**:
- 직원 생산성 (1인당 매출): +18%
- 피크 타임 응대율: 68% → 89% (+31%)
- 직원 만족도: 3.2/5 → 4.1/5 (과부하 감소)

## 5. ROI 상세 계산

### 비용 구조

| 항목 | 월 비용 | 비고 |
|------|---------|------|
| NeuralTwin Professional Tier | ₩490,000 | 기본 구독료 |
| 센서 하드웨어 (6대) | ₩83,000 | 초기 ₩500,000 ÷ 6개월 상각 |
| 설치 및 캘리브레이션 | ₩50,000 | 초기 ₩300,000 ÷ 6개월 상각 |
| **월 총 비용** | **₩623,000** | |

> **참고**: 센서 하드웨어 및 설치비는 초기 비용을 6개월 상각 기준으로 월할 계산했습니다. 7개월차부터 월 비용은 ₩490,000으로 감소합니다.

### 수익 증가

| 항목 | 월 증가액 | 산출 근거 |
|------|----------|-----------|
| 매출 증가분 | +₩100,000,000 | ₩450M - ₩350M |
| 추정 영업이익률 | 15% | 패션 리테일 업계 평균 |
| **월 영업이익 증가분** | **₩15,000,000** | |

### ROI 계산

```
월 순이익 증가 = ₩15,000,000 (영업이익 증가)
월 총 비용     = ₩623,000

ROI = (₩15,000,000 - ₩623,000) / ₩623,000 × 100
    = ₩14,377,000 / ₩623,000 × 100
    ≈ 2,308% (약 23x)

매출 기준 ROI = ₩100,000,000 / ₩490,000
             ≈ 204x (매출 증가 대비 구독료)
```

### 손익분기점

```
월 구독료 ₩490,000 ÷ 영업이익률 15% = 월매출 ₩3,267,000 증가 시 BEP
실제 월매출 증가 ₩100,000,000 → 도입 첫 달부터 BEP 달성
```

## 8. 방법론 및 데이터 신뢰성

### 측정 방법론
- **방문자 수**: WiFi Probe MAC 주소 기반 고유 방문자 카운팅 (MAC 랜덤화 보정 알고리즘 적용)
- **전환율**: POS 거래 건수 / 고유 방문자 수 × 100
- **객단가**: 월 총매출 / POS 거래 건수
- **체류시간**: 최초 감지 ~ 최종 감지 시간 차이 (5분 이상 체류자만 유효)
- **피팅룸 대기시간**: 피팅룸 구역 진입 시점 ~ 피팅룸 입실 시점 차이

### 외부 변수 통제
- 분석 기간 중 대규모 프로모션(세일 등) 기간은 별도 표기
- 계절 변동성은 전년 동기 대비로 보정
- 매장 인근 공사/이벤트 등 외부 요인 발생 시 해당 일자 데이터 주석 처리

### 데이터 수집 준수 사항
- 개인정보보호법 준수: MAC 주소 비가역 해싱 처리
- 매장 내 데이터 수집 고지 안내문 게시
- 수집 데이터 90일 자동 파기 정책 적용
